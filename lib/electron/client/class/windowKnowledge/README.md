# 窗口基础知识
## 窗口相关配置
### 基础属性
#### maxWidth
窗口最大宽度
#### maxHeight
窗口最大高度
#### minWidth
窗口最小宽度
#### minHeight
窗口最小高度
#### resizable
窗口是否改变大小，当为false的时候，窗口将不能改变大小。前面设置的max/min属性没有意义。
#### movable
窗口是否可以移动，当为false的时候，窗口将不能移动。

### 窗口位置
- 默认窗口出现在屏幕的正中间，可以通过设置x和y属性改变窗口出现的位置。
#### x
窗口的x坐标，默认为屏幕中间。
#### y
窗口的y坐标，默认认为屏幕中间。

### 标题栏文本和图标
#### 标题栏文本
- 关于窗口的标题栏，实际上可以通过多个地方进行设置。
- 则涉及优先级问题，具体如下（优先级从上往下）：
  - HTML文档的title
  - BrowserWindow的title属性
  - package.json的name
  - Electron的默认值: Electron

#### 图标
- icon: 设置标题栏的图标，一般是ico图标，也可以是png图标。
```ts
new BrowserWindow({
    //... 
    icon: path.join(__dirname, 'icon.ico')
})
```

### 标题栏、菜单栏和边框
默认创建的窗口是有标题栏、菜单栏和边框的，但是用户也可以通过**frame**属性来决定是否要显示。
- frame: true/false。默认值是true。

## 组合窗口

桌面应用有时候是有多个窗口的，多个窗口彼此之间是相互独立的。也就是说，关闭某一个窗口，其他窗口不会被影响。
但有一些场景中，多个窗口之间存在一定程度的联动。例如两个窗口存在父子窗口关系，一个窗口关闭的时候，另一个窗口也会被关闭。

在Electron中，可以简单实现。Electron提供了父子窗口的概念，通过 **parent** 来指定一个窗口的父窗口。
当形成关系，两个窗口在行为上有一定的联系：
- 子窗口可以相对于父窗口进行定位
- 父窗口在移动的时候，子窗口也会移动
- 父窗口在关闭的时候，子窗口也会关闭
- ......

## 窗口的层级
- 当创建多个窗口的时候，默认情况下，越後面创建的窗口，就在越上层。但是两个窗口是独立的，那么当用户对应的窗口的时候，被点击的窗口会处于最上层。
- 但是在某些场景下，就是希望置顶某一些窗口
  - 创建一个窗口的时候，可以设置 **alwaysOnTop** 为 true，那么这个窗口就会处于最上层。
    - 该配置属性虽然能置顶窗口，但是没有办法进行更颗粒度的设置
  - window.setAlwaysOnTop(flag, level, relativeLevel)：该方法可以设置更加颗粒度的设置
    - flag: 是否置顶窗口
    - level: 置顶级别。一个字符串，指定窗口相对于其他窗口的层级。常用的值包括：**"dock" | "floating" | "main-menu" | "modal-panel" | "normal" | "pop-up-menu" | "screen-saver" | "status" | "torn-off-menu" | undefined**等。这个参数在不同的操作系统上可能会有不同的行为。
    - relativeLevel: 相对置顶级别。一个整数，用于在设置了level的情况下进一步微调窗口层次。
